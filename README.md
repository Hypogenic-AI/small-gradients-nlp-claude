# Small Gradients as a Mechanism for Synthetic Data Issues

## Overview

This project investigates why LLM-generated synthetic data is less sample-efficient than human data and causes model collapse. We hypothesize that synthetic data's low perplexity produces smaller and directionally narrower gradients during fine-tuning, starving the model of learning signal for distributional tails.

## Key Findings

- **Synthetic data has 3x lower median perplexity** than human data (14.6 vs 43.2), confirming LLMs find their own output far more predictable
- **Gradient norms are 15% smaller** for synthetic data (p < 10^-45, Cohen's d = 0.63)
- **Synthetic gradients are directionally collapsed**: only 1-2 PCA components explain 90% of gradient variance, compared to 51+ for human data. This is the key mechanistic finding.
- **1.4-1.8x more synthetic data** is needed to achieve equivalent validation perplexity
- Human and synthetic gradient subspaces have very low overlap (0.13-0.32), meaning synthetic data provides gradient signal in fundamentally different (and narrower) directions

- **Model**: GPT-2 Small (124M params)
- **Dataset**: WikiText-2 (1,581 training sequences of 256 tokens)
- **Synthetic data**: Generated by GPT-2 completing WikiText-2 prefixes (nucleus sampling, p=0.95)
- **Hardware**: NVIDIA RTX A6000 (49 GB)
